## 已确认的Bug（1个）

### Bug 1 — P2: `parse_char_const` 缺少八进制转义支持

**位置**: `expr.c:124-156` `parse_char_const()` 函数

**现象**: 
- `'\n'`, `'\t'`, `'\0'`, `'\x41'` 都正确处理
- `'\101'` (八进制) 未处理，返回字符 `'1'` 而非 `'A'`

**原因**: 
- `parse_char_const` 只处理了 `\n`, `\t`, `\r`, `\\`, `\'`, `"`, `\0`, `\a`, `\b`, `\f`, `\v`, `\x`
- **缺少八进制转义**: `\ddd` (1-3位八进制数字)

**修复方案**:
在 `\x` case 之前添加八进制处理：
```c
case '0': case '1': case '2': case '3':
case '4': case '5': case '6': case '7': {
    uval v = 0;
    int count = 0;
    while (count < 3 && *sp >= '0' && *sp <= '7') {
        v = v*8 + (uval)(*sp - '0');
        sp++;
        count++;
    }
    return (ival)v;
}
```

**影响**: 代码中使用八进制转义的字符常量在 `#if` 表达式中值不正确
